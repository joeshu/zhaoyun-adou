**当前阶段瓶颈**（结合实界面截图重定）：不是"能不能玩"，而是 **"好不好玩 / 耐不耐玩"**。三个最显眼的短板：
1. **抉择密度不足**——核心 loop 成立，但自动战斗 + 全正收益遗物，让"动脑子"的瞬间太少。
2. **表现力有骨架缺血肉**——`animT` 弹入、屏震、粒子、底衬都已就位，但缺**统一缓动框架**与**高光时刻演出**，爽感没被"放大"。

## 1. 玩法（Gameplay）

### 1.1 🔴 遗物系统加权衡  `[核实: modes.js:99-104]`
现状：每 5 波三选一，6 个选项**全是正收益**（伤害+25% / 攻速+15% / 血量+20% / 馒头+30 / +50 / 回血3），且每波完全相同。
问题：这不是选择，是"永远拿 % 属性最强的那个"，零决策。
方案：
- 注入**双刃遗物**：`背水`（全军伤害+40%，阿斗血量−2）、`劫粮`（下一波馒头×2，敌人数+30%）、`孤注`（攻速+30%，但本波不产金币）。
- **随波次放大**：第 5 波 +20% → 第 30 波 +45%（倍率存表，便于 tuning）。
- 保底：每轮至少保留 1 个纯增益，避免新手误选爆炸。
验收：玩家在"资源 vs 风险"间出现犹豫；录像显示双刃选项被选率 > 25%。

### 1.2 🔴 战斗中给一个"主动决策"  `[核实: 技能自动释放 battle.js castSkill]`
现状：武将技能由 `castSkill` 到点自动放，玩家全程看戏。
方案：每波允许**手动触发 1 次武将大招**（HUD 显示 CD/就绪），贴合"赵云七进七出"幻想，把被动观看变主动出手。
验收：手动大招使用率 > 60%；手动 vs 自动胜率差 < 10%（避免变成强制操作）。

### 1.3 🟠 难度曲线建模  `[核实: data.js boss 200→2600]`
现状：关卡 1→37，BOSS 血量 200（张梁）→ 2600（司马懿），13× 跨度，但中间小怪/波次的成长是否几何递增未建模。
方案：建立 `敌人HP = base × growth^关卡` 曲线表，配合我方 `单位DPS = f(等级,觉醒,羁绊,遗物)` 做纸面模拟，定义"第 N 关目标通关率 60%"再回填数值。
验收：任意连续 5 关的"理论通关率"落在 45%–75% 区间，无断崖。

### 1.4 🟠 模式差异与"心"机制  `[核实: modes.js 各模式 mix/hp 公式]`
现状：7 种模式公式各异，但玩家**感知差异**可能仅停留在"出怪不同"。建议每个模式有一个**标志性机制提示**（如护送模式突出"护盾=可掉"、残局突出"残阵翻盘"），并在选模式页用一句话讲清"这局爽点是什么"。
验收：选模式页每个模式有 ≤12 字卖点 + 1 张示意图。

### 1.5 🟠 新手前 60 秒 Onboarding  `[核实: 成就 first_summon/first_hero/first_win]`
方案：落地 onboarding checklist——核心动词（抽/放/合）30 秒内上手、首胜必得（第 1 关不用懂遗物/锻造）、每个新机制在低 stakes 场景引入、首局结束给一个"再玩一局"钩子。
验收：无设计师协助下首胜率 > 90%。

### 1.6 🟡 数值平衡：觉醒 / 羁绊 / 武器  `[核实: merge.js awaken 1.3³≈2.2×]`
现状：觉醒 ≤2.2× 天花板合理，不会 runaway；武器加成 8%–40% 区间合理。
方案：把武器/羁绊/觉醒的乘区做成一张**可调杠杆表**（目前散落在 merge.js 各 `if w===` 分支），便于后续平衡。

## 2. 画面（Visuals / Juice）

### 2.1 🔴 统一缓动框架  `[核实: 仅 animT 弹入 battle.js/actions.js]`
现状：有 `animT`（放置/合成/受击 0.18–0.3s 缩放），但**无统一 easing/tween**，移动无插值、UI 无过渡、血量条瞬变。
方案：引入一个极轻量 tween 工具（`tween(obj, key, to, dur, ease)` + 每帧 `updateTweens(dt)`），缓动函数用 `easeOutBack`/`easeOutCubic`。这是后续所有"演出"的地基。
验收：所有位移/缩放/透明度过渡走 tween，无硬编码线性跳变。

### 2.2 🟠 单位表现分层  `[核实: drawBase 底衬已加 ui_battle.js]`
现状：`drawBase` 已按兵种加圆/盾/菱/方底牌（好）。建议再加两层：
- **阵营光晕**：玩家侧暖色描边、敌方冷色，一眼分敌我。
- **状态环**：单位脚下细环显示 眩晕/减速/狂暴 等异常状态（颜色编码）。
验收：不读字也能分清敌我 + 当前异常状态。

### 2.3 🟠 技能特效体系化  `[核实: fx 机制已复用]`
现状：武将技能特效零散。建议为每个武将大招配**专属视觉签名**（如赵云=青色突刺残影、张飞=紫色震波），复用现有 `fx` 机制但统一参数接口。
验收：6+ 主将大招有可辨识视觉，玩家能"看招识将"。

### 2.4 🟠 高光时刻演出  `[核实: 觉醒仅橙色小圆点]`
- **觉醒**：单位放大弹一下 + 金色光环 + 横幅"XX 觉醒！"，让 2.2× 数值暴涨被"感受到"。
- **暴击 / 羁绊激活 / BOSS 破防**：分别对应专属闪白 / 羁绊连线高亮 / 破甲碎裂粒子。
验收：关键事件有 ≥0.4s 的专属演出，且不与屏震叠加到眩晕。

### 2.5 🟡 战场氛围  `[假设: 背景近乎纯色]`
方案：出怪口做"敌军涌出"的暗红裂隙、阿斗位做"营寨"底纹、地面做轻微格线，用低饱和避免抢单位。
验收：战场有纵深感但不喧宾夺主（单位辨识度不降）。

> 截图补充：战场里**最抢眼的是"荒"格**（占屏 ~40%，黄色填充 + 大字"限20"），反而把单位淹没了。**先把"荒"格视觉降权（§4.3）** 再谈氛围；否则加底纹也是"在噪声上叠噪声"。

## 3. UI（交互与反馈）

### 3.1 🟠 按钮/状态反馈
现状：`btn()` 已有 disabled 态。建议补 **按压缩放**（按下缩 0.94 + 释放回弹，走 2.1 的 tween）+ **音效联动**（见 §6）。
验收：所有可点元素有按下/抬起/禁用三态视觉。

### 3.2 🟠 飘字系统化  `[核实: fl() 飘字已普遍使用]`
现状：`fl()` 飘字基础好。建议分类型配色/动效：伤害（白）、暴击（红+放大）、馒头（棕）、治疗（绿）、警告（黄闪）。建立 `floatText(type, ...)` 统一入口。
验收：不同事件飘字一眼可辨，不再全是同色。

### 3.3 🟠 数值可读性
现状：单位无血条，buff 仅靠"橙点"。建议：
- 单位头顶**细血条**（敌我同款，敌方红、友方绿）。
- **buff 图标行**：觉醒/武器/羁绊/异常以小图标横排在单位下方，替代"数圆点"。
验收：不点开面板也能读出"这单位现在强在哪、弱在哪"。

### 3.4 🟡 顶栏功能可发现性
现状：速度/撤销/下一波预告已加，但新玩家可能不知"撤销能防手滑"。建议首次进入给 1 次轻提示气泡。
验收：新玩家首局撤销使用率 > 15%。

### 3.5 🟠 资源图标化（菜单 + 战斗通用）  `[核实: 截图]`
现状：菜单 `金 10000 / 材料 2`、战斗 `馒30 / 金9255` 都是纯文字。
方案：引入 2–3 个 8–12px 的**小图标**（金币圆/馒头/材料方/护盾菱/武将印），所有显示走统一 `{icon, name, value}` 组件。零资源：用 emoji 或 Canvas 画的极简符号即可。
验收：玩家扫一眼就能识别"哪个是馒头、哪个是金币"，不用读字。

### 3.6 🟠 "当前"按钮消歧（截图问题）  `[核实: 截图]`
现状：底栏"当前"两字，无图标，玩家不知道是"当前选中单位"还是"当前波次"。
方案：改为更直白的动词或加图标，例如"查看选中"或直接改为"详情"；或彻底拿掉（如果"当前"的功能已经被其他 UI 覆盖）。

## 4. 布局（Layout）

### 4.1 ✅ 栅格已规整  `[核实: W375 H667 TOP32 CELL48, 5列, 10格合成栏]`
棋盘点位用 `MAPS[].ROWS_P/ROWS_E/COLS` 统一管理，合成栏 `BAR_COLS=[45,113,181,249,317]` 等距——地基干净，不用动。

### 4.2 🔴 顶栏拥挤 / 文字被遮挡（截图 bug）  `[核实: 截图]`
现状：375px 顶栏要塞 `馒30 / 金9255 / 第13关·第7波 / ×1 / II / 音 / 菜单` 7+ 元素，`第13关·第7波` 被速度/暂停按钮**部分遮挡**，玩家读不到当前关/波。
方案：
- 关/波信息升级为"主信息位"（左侧大字号），速度/暂停/音/菜单收进**右侧紧凑 4 颗圆按钮**。
- 资源（馒/金）保留左侧但**带小图标**（见 §3.5）。
- 加 `Game.raf` 之前先 layout 测一下文字宽度，避免再压字。
验收：375px 宽度下顶栏无任何元素重叠；关/波信息一眼可读。

### 4.3 🟠 棋盘"荒"格视觉权重过大（截图问题）  `[核实: 截图]`
现状：棋盘约 40% 面积是"荒"+"限20"字样，玩家第一眼被它吸走，反而看不清已部署单位。
方案：
- "荒"格改为**细线 + 角标小字"20"**，不抢主视觉。
- 鼠标/触屏长按"荒"格弹 tooltip："开荒需 20 馒头"。
验收：首眼视觉焦点是已部署单位 + 阿斗，不是"荒"格。

### 4.4 🟠 拖动时高亮落点/合成对
现状：合成栏在底部、棋盘在中部。建议在"拖动单位"时，棋盘格**高亮可落点**，合成栏格**高亮可合成对**，降低误操作。
验收：拖动过程中落点/合成预览清晰，误放率下降。

### 4.5 🟡 双"阿斗"标签消歧（截图问题）  `[核实: 截图]`
现状：棋盘上中、下中各一个"阿斗 ♥3"，玩家会困惑"哪个是我"。
方案：明确敌/我——用箭头/颜色区分"敌方目标线 → 我方阿斗"，或把"敌方出怪口"改为"敌军涌出处"标签，避免与阿斗撞名。

### 4.6 🟡 大屏 / 横屏适配
现状：`ui.js:686` 用 `scaleF*dpr` **整体放大**画布，大屏只是"放大像素"，布局仍是 375 逻辑宽。
方案：把布局参数改为**基于逻辑宽高百分比**而非硬编码坐标（或引入一个 `LAYOUT` 适配层），让 iPad/横屏能 reflow 而非拉伸。
验收：在 768×1024 与横屏下，关键元素不溢出、不比例失调。

## 6. 音频（补遗）

### 6.1 ✅ SFX 已具雏形  `[核实: js/audio.js Web Audio 合成短音效, 零资源]`
用 Web Audio 程序化合成（无音频文件依赖），无 AudioContext 时静默降级——工程上很聪明。

### 6.2 🟠 缺 BGM 与音频设置入口
现状：grep 无 `bgm/music`，仅有短音效；`isMuted()` 暗示有静音标记，但**设置入口可发现性未知**。
方案：
- 加 1–2 段**程序化环境 BGM**（可用 Web Audio 做简单循环 pad/鼓点，仍零资源）。
- 设置面板加 **音效/音乐 独立开关 + 音量条**。
验收：玩家能分别控制 SFX/BGM；静音状态持久化。

### 6.3 🟡 音效事件审计

## 8. 量化验收指标（成功标准）

### 8.1 截图驱动的硬指标（Phase 0 验收用）
- **顶栏**：375px 下关/波信息不被任何控件遮挡，文字可读距离 ≥ 1m。

- **Onboarding**：无协助首胜率 > 90%；核心动词 30 秒内理解。
- **抉择密度**：遗物双刃选项被选率 > 25%；手动大招使用率 > 60%。
- **难度曲线**：任意连续 5 关的"理论通关率"落在 45%–75% 区间。

### 8.3 表现力 / 音频 / 适配
- 关键事件（觉醒/暴击/BOSS 破防）有 ≥0.4s 专属演出，且不与屏震叠加眩晕。
- SFX/BGM 独立可控；静音持久化。
- 768×1024 与横屏下无溢出/比例失调。
- 性能：保持前轮优化后的帧率预算（战场 HUD 阴影收敛、热路径零分配）。
